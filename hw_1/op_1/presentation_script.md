# Seam Carving 汇报稿

大家好，我汇报的是这次作业中实现的内容感知图像缩放算法，也就是 Seam Carving。

这项算法的目标，不是像普通缩放那样把整张图统一拉伸或者压缩，而是尽量保留图像中的主体结构，把不重要的区域优先删除或者复制。这样做的好处是，在缩小或者放大图像时，像人物、岩石、建筑轮廓这类重要内容能尽量保持自然，而不是被整体挤扁。

我的实现分成三个核心部分：能量函数设计、最优 seam 搜索，以及 seam 的删除和插入。

第一部分是能量函数。这里我没有直接用最常见的一阶梯度，而是按照作业要求，采用了基于 Laplacian 的能量。具体做法是，对 RGB 三个通道分别做一个 3 乘 3 的离散拉普拉斯卷积，再把三个通道响应的平方加起来，得到每个像素的能量值。
先给整个图像的四条边各自补上一行，用来做边缘的计算。由于对每个点计算能量的方法都是一样的，所以可以直接把整张图像做一个偏移，然后乘卷积核对应的元素，再把结果加起来求平方，得到整个图像上每一点的能量。

第二部分是最优 seam 的搜索。对于一条竖直 seam，它要求每一行只经过一个像素，并且相邻两行的位置偏移不超过 1。给出两个矩阵：cost和parent，分别代表到达该点需要的最小能量，以及对应路径的上一层坐标。为了找到总能量最小的一条路径，我用了动态规划。从第一行往下规划，到达第一行的最小能量，自然是它本身的能量；到达第二行的最小能量，只需要考虑它的左上、正上、右上三个方向的能量，哪个最小就可以了，把那个最小的能量加上第二行对应像素的能量，再把最小能量的位置写入parent矩阵对应位置。以此类推直到最后一行。
接下来，取最后一行累计能量最小的像素，根据parent矩阵一直回溯到第一行，返回这条路径的各点坐标。

第三部分是 seam 的实际操作。缩小图像时，创建一个mask布尔矩阵，初始值全是True，代表每个位置像素是否保留。把每个需要删去的位置改成False，然后按照mask矩阵，把图像写入新的少一列的图像。

另外，为了统一横向和纵向的处理逻辑，我没有专门写两套算法，而是通过图像转置，把横向缩放转换成纵向 seam 处理，这样代码结构更简洁，也更容易维护。

从实验结果来看，缩小图像时，这个方法比普通缩放更能保留主要景物的结构关系；放大图像时，批量 seam 插入也减轻了大面积平坦区域被重复拉伸的问题。当然，这个算法也有局限，比如当目标缩放比例过大时，局部失真还是会比较明显，而且每次操作后都要重新计算能量图，所以运行时间相对较长。

总的来说，这次作业让我比较完整地实现了 Seam Carving 的基本流程，也加深了我对“内容感知”这一思想的理解。它的关键并不是简单做几何缩放，而是先定义什么内容重要，再把这个重要性转化成一个可优化的路径搜索问题。谢谢大家。
